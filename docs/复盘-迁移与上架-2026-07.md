## 二、国际支付方案 B（Payoneer）

**背景**：Lemon Squeezy 账户仍在审核，海外无法自助收款。Payoneer 无面向独立开发者的自助 checkout API，故采用**人工收款 + 兑换码**方案。

**实现**：
- 服务端 `.env` 新增 `SUPPORT_EMAIL`（客服邮箱，单一配置源）、`PAYONEER_URL`（可选）。
- `/api/pay/providers` 暴露 `payoneer:!!SUPPORT_EMAIL` 与 `support{email,payoneerUrl}`。
- 购买页 International 标签：LS 未配置时启用 Payoneer 联系购买流程（步骤说明 + 一键发邮件 + 复制邮箱 + 可选 Payoneer 付款按钮）。
- 扩展账号面板显示客服入口。
- **优先级**：`intlProvider()` 优先 Lemon Squeezy，其次 Payoneer——LS 审核通过配好后自动切回，无需改代码。

**闭环**：海外用户邮件联系 → 你用 Payoneer「Request a Payment」发付款链接 → 到账后在 `/admin` 生成兑换码发给用户 → 用户在扩展内兑换开通。

**国际定价**（美元参考价，实际以 Payoneer 付款请求为准）：月卡 $2.99 / 季卡 $6.99 / 年卡 $11.99。

## 三、服务器迁移：GCP → 甲骨文云（Oracle Cloud）

### 迁移前后对照
| 项目 | 旧（GCP）| 新（Oracle）|
|------|---------|------------|
| 公网 IP | 34.64.114.22 | **168.138.42.168** |
| 机型 | — | VM.Standard.E2.1.Micro（x86，1GB）|
| 对外地址 | http://34.64.114.22:8787（明文IP）| **https://api.kpl.us.kg**（HTTPS 域名）|
| 反代/证书 | 无 | Caddy 自动 Let's Encrypt |

### 关键步骤与踩过的坑
1. **双层防火墙**（甲骨文特有大坑）：云控制台安全列表放行 80/443 + 系统内 `iptables -I INPUT ... --dport 443/80 ACCEPT` 并 `netfilter-persistent save`。两层都开才通。
2. **swap**：1GB 内存机器，已有 2GB swap，npm 编译 better-sqlite3 无压力。
3. **数据迁移的核心坑——SQLite WAL**：主库 `data.db` 仅 4KB，真实数据在 `data.db-wal`(1.2M) 里。
   - ❌ 直接 `cp data.db` 只拷到空壳（丢数据）。
   - ✅ 正解：`sqlite3 data.db ".backup 'data-migrate.db'"`（合并 WAL），备份变 596K，`SELECT count(*) FROM users` = 4，两边一致才算成功。
4. **.env 迁移**：`JWT_SECRET` 原样保留（否则用户全登出）；`PUBLIC_BASE_URL` 改为 `https://api.kpl.us.kg`；新增 `SUPPORT_EMAIL`；支付宝配置照搬。
5. **文件传输**：GCP 浏览器 SSH 下载需先 `sudo cp` 到有权限目录；scp 上传 Windows 私钥要 `icacls` 收紧权限（否则 "UNPROTECTED PRIVATE KEY"）。
6. **PM2**：`pm2 start src/index.js --name pgpro` + `pm2 save` + `pm2 startup`（root 身份会自动执行 systemctl enable，无需手动复制命令）。

### 域名与 HTTPS
- 域名 `kpl.us.kg` 主域名已被其它服务占用（多条 A 记录），故 PanGrab 专用子域名 **`api.kpl.us.kg`**。
- Cloudflare 加 A 记录 `api` → `168.138.42.168`，**代理状态必须「仅 DNS」（灰云）**，否则 Caddy 无法签发证书。
- Caddy 配置：`api.kpl.us.kg { reverse_proxy localhost:8787 }`。
- 验证：`curl https://api.kpl.us.kg/api/health` → `{"ok":true}` ✅

## 四、Chrome 网上应用店上架

### 上次拒审原因（务必记住）
- 版本 **1.18.2**（无会员版），2026-06-30 被拒。
- 违规类型：垃圾内容和商店中的排名，参考 ID **Yellow Argon**。
- 根因：**产品说明关键字堆砌**——描述里罗列了 27 个网盘名称（百度/阿里/夸克…Google Drive/Dropbox/MEGA…）。
- 整改：**删除网盘名称清单**，改用自然描述，最多举 2–3 个例子。

### 本次会员版（2.0.0）上架要点
1. **版本号**：升到 `2.0.0`（必须高于 1.18.2，否则版本冲突被拒）。
2. **商品详情**：三语言（英/简/繁）说明**都删掉网盘清单**，用干净文案；至少 1 张 1280×800 截图。
3. **隐私权规范**：单一用途 + **逐条权限理由**（含新增 `alarms`：定时云同步每30分钟）+ 数据用途声明（邮箱/认证/网站内容）+ 三项合规认证 + 隐私政策 URL `https://api.kpl.us.kg/privacy`。
4. **分发**：定价选 **「包含应用内购买」**（有付费会员；谷歌自带支付已于 2020 停用，用自有支付合规）。
5. **CRX 上传公钥**：可选安全功能，**不启用无影响**，跳过。
