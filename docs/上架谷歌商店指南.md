## 一、前置准备

1. **开发者账号**：注册 [Chrome Web Store 开发者后台](https://chrome.google.com/webstore/devconsole)，需一次性缴 **$5** 注册费。
2. **验证联系邮箱**：后台 Account → Contact email，填并验证。**建议与 `.env` 的 `SUPPORT_EMAIL` 用同一个**，用户与谷歌看到的客服邮箱一致。
3. **确认后端在线**：
   - `https://api.kpl.us.kg/api/health` → `{"ok":true}`
   - `https://api.kpl.us.kg/privacy` 能打开
   - `https://api.kpl.us.kg/` 落地页能打开

## 二、打包扩展（关键：ZIP 结构别搞错）

扩展目录是 `extension/`。上传的 ZIP **必须让 `manifest.json` 位于压缩包根目录**。

- ✅ 正确：进入 `extension` 文件夹 → 全选里面的内容（`manifest.json`、`icons`、`_locales`、`src`）→ 压缩成 zip
- ❌ 错误：把 `extension` 这个文件夹整个压进去（会导致 manifest 在子目录里，审核报错 "Manifest file is missing or unreadable"）

**Windows 操作**：打开 `extension` 文件夹 → `Ctrl+A` 全选 → 右键 → 压缩为 ZIP 文件。

> 打包前确认 `pro-config.js` 里 `apiBase` 是 `https://api.kpl.us.kg`（已是）。

## 四、权限理由（Privacy practices → 逐条填写）

以下均为代码中**实际使用**的权限，可直接填入对应理由框：

| 权限 | 填写的理由 |
|------|-----------|
| `storage` | 在本地保存用户的登录令牌、收藏列表与设置。 |
| `activeTab` | 用户点击扩展弹窗时，读取当前活动标签页以扫描该页的网盘/磁力链接。 |
| `scripting` | 从弹窗向当前页面注入脚本，提取页面中的分享链接与提取码。 |
| `contextMenus` | 提供右键菜单，便于用户快速收藏选中的链接。 |
| `notifications` | 向用户提示操作结果（如收藏成功、链接失效）。 |
| `alarms` | 定时触发后台云同步任务（每 30 分钟）。 |
| **主机权限 `<all_urls>`** | 网盘分享链接可能出现在**任意网站**，需要在所有页面运行检测脚本以识别链接与提取码。这是本扩展的核心功能。 |

> `<all_urls>` 属于广权限，审核会重点看这条理由——务必讲清"链接可能出现在任意站点"。

## 五、数据用途声明（Data usage）

如实勾选与声明：

**收集的数据类型**：
- **个人身份信息**：电子邮箱（用于账号）
- **身份验证信息**：密码（用于账号登录，服务端以 bcrypt 哈希存储）
- **网站内容**：用户主动收藏的链接/网址（仅在开启云同步时上传到其本人账号）

**三项合规认证（必须勾选“是”/符合）**：
- ✅ 不会将用户数据出售或转让给第三方（支付第三方除外，且仅用于处理交易）
- ✅ 不会将数据用于与扩展核心功能无关的用途
- ✅ 不会将数据用于判定信用/放贷

**隐私政策 URL**：`https://api.kpl.us.kg/privacy`

## 七、付费会员声明（Distribution 标签 → 包含应用内购买）⭐

自 2020 年起 Google 已**关停 Chrome 商店自带支付系统**，扩展不能再用谷歌收银台收费。付费功能须用**自有支付**（本项目用支付宝 / Payoneer / 兑换码 + 自有后端），这是允许的做法。

由于 PanGrab Pro 含付费 Pro 会员（用第三方/自有支付提供订阅/付费功能），在开发者后台需声明：

1. 进入你的扩展 → **Distribution（分发）** 标签
2. 定价类型选 **「Contains in-app purchases（包含应用内购买）」**，不要选「Free」
3. 保存后，商店详情页会显示「应用内购买 / in-app purchases」徽章

> - 扩展本身仍免费安装（店内不显示价格标签），实际收费通过自有后端完成，合规。
> - 不需要接入谷歌支付（该系统已停用），也没有别的"商用开关"。
> - 务必保持定价透明——购买页与隐私政策已覆盖。

## 八、审核注意事项 / 风险点

1. **广权限审核更严**：`<all_urls>` + 全站内容脚本会触发人工审核，理由要充分（见第四节）。
2. **内容合规**：商店描述、截图保持**中性、专业**，聚焦"网盘链接管理工具"这一通用功能，遵守 [Chrome 网上应用店政策](https://developer.chrome.com/docs/webstore/program-policies)。避免出现违规或成人相关内容表述。
3. **后端必须在线**：审核员会实际安装测试，登录/云同步依赖 `https://api.kpl.us.kg`，提交期间保持后端可用。
4. **首次审核**通常 1–3 个工作日，广权限可能更久。
